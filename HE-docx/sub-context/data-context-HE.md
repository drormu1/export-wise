## הפרדת שכבות (חובה)
1. **SQL Database** = אמת עסקית.
2. **Vector DB** = ארטיפקטים סמנטיים שמופקים מ-SQL.
3. **LLM** = ניתוח מייעץ בלבד.

## SQL Database כמקור אמת
בסיס הנתונים ה-SQL הוא המקור הסמכותי לכל הישויות העסקיות וההחלטות.
בכל קונפליקט בין SQL לשכבות אחרות, יש להעדיף את נתוני SQL.

## פרופיל ספק בסיס נתונים
- ברירת מחדל ב-POC הנוכחי: SQLite
- ספק אופציונלי/יעד למעבר עתידי: SQL Server
- שכבת הנתונים צריכה להישאר אגנוסטית לספק (`sqlite | sqlserver`).

## היקף Seed בסיסי (POC)
טעינת ה-seed הראשונית חייבת לכלול בדיוק:
- `manufacturers`: 200 רשומות
- `countries`: 200 רשומות
- `products`: 200 רשומות

רשומות אלו צריכות לכלול רק שדות הכרחיים ל-POC תוך שמירה על ריאליזם.

## טבלת החלטות רישוי (נתונים מיוצרים)
החלטות רישוי דמויות-היסטוריה שנוצרות נשמרות ב-`licenses`.

שדות מינימליים:
- `manufacturer_code`
- `product_code`
- `country_code`
- `decision_status`
- `limitations`

מפתח ראשי:
- מפתח מורכב: (`manufacturer_code`, `product_code`, `country_code`)

כוונת מפתחות זרים:
- `manufacturer_code` מפנה ל-`manufacturers`
- `product_code` מפנה ל-`products`
- `country_code` מפנה ל-`countries`

## כלי יצירת Mock מבוסס Iteration
כלי צדדי מייצר רשומות `licenses` נוספות באמצעות:
- קלט: `iteration`

כללים:
- שימוש רק בקודים קיימים מטבלאות הבסיס ב-SQL.
- יצירת תוצאות החלטה מגוונות וריאליסטיות.
- הכנסת מפתחות מורכבים ללא התנגשות בלבד.
- דילוג על מפתחות קיימים בלי להפיל את כל הריצה.
- אפשרות להרצות חוזרות להעשרה רציפה של הנתונים.

פלטים מומלצים מהריצה:
- `generatedCandidatesCount`
- `insertedCount`
- `skippedConflictsCount`
- `failedCount` (אם יש)

## סנכרון SQL -> Vector (חוזה ברמת על)
לאחר כל ריצת העשרה ל-SQL מבוססת iteration שהצליחה, ריצת אינדוקס ל-Vector היא חובה.
התנהגות אינדוקס מפורטת, גבולות בעלות, טיפול בכשלים וחוזי שליפה מוגדרים ב:
- `sub-context/vector-context.md`

## כללי צריכת נתונים ב-LLM
- ה-LLM מקבל הקשר מתוצרי שליפה (SQL + Vector).
- אין להתייחס ל-LLM כאל אחסון או זיכרון היסטורי.
- כל המלצה חייבת להיות מבוססת על מקרים שנשלפו.
- אם הראיות חלשות, התשובה חייבת לציין אי-ודאות במפורש.

## בעלות על מסמכי חיפוש
רשומות `licenses` הן המקור העסקי ליצירת מסמכי retrieval.
מבנה מסמכים וכללי אינדוקס מנוהלים ב:
- `sub-context/vector-context.md`
