# תמיכת החלטה בייצוא מזון באמצעות AI

## חזון הפרויקט

הפרויקט הוא מערכת תמיכת החלטה מבוססת AI לאישורי ייצוא מזון.

המטרה אינה להחליף את הוועדה.

המטרה היא לסייע לחברי הוועדה באמצעות ניתוח החלטות קודמות, רגולציות ומידע על מוצרים לפני תהליך הבדיקה הרשמי.

ה-AI צריך תמיד לפעול כיועץ.

ההחלטה הסופית תמיד שייכת לוועדה אנושית.

כיוון נוכחי:

Backend

- Node.js
- TypeScript

Database

- SQLite (ברירת מחדל ל-POC הנוכחי)
- SQL Server (תאימות אופציונלית/יעד)

AI

- Claude
- GPT
- Gemma

חיפוש סמנטי

המימוש צריך לאפשר הוספת Vector Database בהמשך  
(Qdrant, PostgreSQL + pgvector, Azure AI Search או Elasticsearch).

Frontend

כנראה שיפותח בהמשך Frontend וובי באמצעות Angular.

אין להניח הנחות לגבי ארכיטקטורת ה-frontend אלא אם התבקשת.

# הנחיות AI

ה-AI אינו מקבל ההחלטה.

ה-AI צריך:

- למצוא החלטות היסטוריות דומות.
- להשוות בין מקרים.
- להסביר דמיון.
- להסביר הבדלים.
- להדגיש סיכונים.
- להציע תנאי אישור.
- לייצר המלצות מובנות.

לעולם אל תמציא עובדות.

יש להפריד תמיד בין:

- Facts
- Assumptions
- AI reasoning

כל המלצה צריכה להתייחס, ככל האפשר, להחלטות היסטוריות תומכות.

# מקור נתונים

הגרסה הראשונה של הפרויקט לא תשתמש בנתוני ייצור.

יש ליצור בסיס נתונים דמו ריאלי באמצעות SQLite בשלב ה-POC הנוכחי.
יש לשמור על תאימות לשכבת נתונים של SQL Server לצורך מעבר/הרצה עתידיים.

יש לאכלס אותו באמצעות קובץ Seed.

נתוני ה-Seed צריכים לדמות כמה שנות החלטות ועדה.

בסיס הנתונים צריך להכיל מספיק נתונים כדי להדגים חיפוש סמנטי.
